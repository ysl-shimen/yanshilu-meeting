# APP会议文档

---

## 1 会议主要功能

- 启/停摄像头
- 启/停麦克风
- 共享屏幕
- 录制/停止录制
- 视频回放
- 离开/结束会议等

>  移动端，要支持触屏，拖动等常用功能；

---

## 2 会议流程

### 2.1 主要流程

学生购课成功 -> 平台生成会议链接 -> 点击会议链接开始上课 -> 上课完成，结束会议 -> 点击会议链接查看回放

### 2.2 详细流程

1. 学生或老师访问会议链接：

   1.1 需要验证用户身份，身份不匹配或者token无效等，拒绝进入会议；

   1.2 获取会议信息，检查会议状态，如果会议状态是已完成，跳转到视频回放页面，并调用获取回放视频列表接口，支持观看视频；

   1.3 如果会议状态是待开始，则在进入会议时，更新会议状态为：进行中，如果会议中只有一个人，调用开启录制接口，开启视频录制；

2. 结束上课，会议结束：

   2.1 点击结束会议，调用结束录制接口，结束录制；

   2.2 直接离开会议，如果会议内还有人，无需其他流程；如果会议内没有人了，还要调用结束录制接口，结束录制；

   2.3 结束录制后，阿里云 RTC 通过 Webhook 回调通知后端，后端自动更新录制文件信息到数据库，防止网络等原因，也主动查询录制信息；

### 2.3 注意事项

- 在app上点击会议链接，跳转到app的会议页面；在浏览器访问会议链接，跳转到Web端的会议页面；

### 3.2 会议

#### 3.2.1 加入会议

> **接口使用场景：** 用户成功进入 RTC 频道后调用。后端自动处理：课时开始时间记录、录制任务查询与去重、启动云端录制、创建录制记录等多步操作，前端只需调用一次。
>
> **请求URL示例：** `https://meeting.yanshilu.com/api/meeting/join`
>
> **请求方式：** POST
>
> **请求参数：**
>
> | 参数名 | 类型 | 必填 | 说明 |
> | --- | --- | --- | --- |
> | channelId | string | 是 | 频道ID |
>
> **返回参数：**
>
> | 参数名 | 类型 | 说明 |
> | --- | --- | --- |
> | code | number | 状态码，200为成功 |
> | data | object | 返回数据 |
> | data.taskId | string \| null | 录制任务ID；录制功能关闭或启动失败时为 null |
> | data.isRecording | boolean | 是否已开启录制 |
> | message | string | 返回消息 |
>
> **请求参数示例：**
>
> ```json
> {
>   "channelId": "1770907673327"
> }
> ```
>
> **返回参数示例（成功，录制已开启）：**
>
> ```json
> {
>   "code": 200,
>   "data": {
>     "taskId": "3e26da51-2b53-4c7d-a6aa-7aaa9e93e2bf",
>     "isRecording": true
>   },
>   "message": "success"
> }
> ```
>
> **返回参数示例（成功，录制功能已关闭）：**
>
> ```json
> {
>   "code": 200,
>   "data": {
>     "taskId": null,
>     "isRecording": false
>   },
>   "message": "success"
> }
> ```
>
> **返回参数示例（失败）：**
>
> ```json
> {
>   "code": 500,
>   "data": null,
>   "message": "startCloudRecord error: ..."
> }
> ```

#### 3.2.2 离开会议

> **接口使用场景：** 用户正常点击"退出"按钮离开会议时调用；页面关闭/刷新等异常退出时通过 `navigator.sendBeacon` 调用（Content-Type 为 text/plain）。若当前用户是最后一个，后端自动停止录制，录制文件信息由阿里云 Webhook 回调异步更新。
>
> **请求URL示例：** `https://meeting.yanshilu.com/api/meeting/leave`
>
> **请求方式：** POST
>
> **请求参数：**
>
> | 参数名 | 类型 | 必填 | 说明 |
> | --- | --- | --- | --- |
> | channelId | string | 是 | 频道ID |
> | userId | string | 是 | 离开的用户ID |
> | taskId | string | 否 | 录制任务ID；无录制任务时可不传或传空 |
> | userCount | number | 否 | 离开时频道内的用户数（含自身），后端据此判断是否为最后一人；不传时默认按最后一人处理 |
>
> **返回参数：**
>
> | 参数名 | 类型 | 说明 |
> | --- | --- | --- |
> | code | number | 状态码，200为成功 |
> | data | null | 无返回数据 |
> | message | string | 返回消息 |
>
> **请求参数示例：**
>
> ```json
> {
>   "channelId": "1770907673327",
>   "userId": "40440",
>   "taskId": "3e26da51-2b53-4c7d-a6aa-7aaa9e93e2bf",
>   "userCount": 1
> }
> ```
>
> **返回参数示例（成功）：**
>
> ```json
> {
>   "code": 200,
>   "data": null,
>   "message": "success"
> }
> ```
>
> **返回参数示例（失败）：**
>
> ```json
> {
>   "code": 400,
>   "data": null,
>   "message": "channelId 和 userId 不能为空"
> }
> ```

#### 3.3.3 阿里云录制 Webhook 回调

> **接口使用场景：** 由阿里云 RTC 服务器主动调用，前端不调用此接口。需在阿里云 RTC 控制台配置回调地址为 `https://meeting.yanshilu.com/api/record/callback`。接收录制开始（2000）、录制成功（2001）、录制失败（2002）事件，自动更新数据库中的录制状态和文件信息。
>
> **请求URL示例：** `https://meeting.yanshilu.com/api/record/callback`
>
> **请求方式：** POST
>
> **请求头：**
>
> | 参数名 | 类型 | 说明 |
> | --- | --- | --- |
> | dingrtc-signature | string | 阿里云 RTC 签名，后端自动验证，验证失败返回 403 |
> | Content-Type | string | application/json |
>
> **请求参数（阿里云推送的 body）：**
>
> | 参数名 | 类型 | 说明 |
> | --- | --- | --- |
> | eventType | string | 事件类型：`001`（回调验证）/ `2000`（录制开始）/ `2001`（录制成功）/ `2002`（录制失败） |
> | eventData | object | 事件数据 |
> | eventData.channelId | string | 频道ID |
> | eventData.taskId | string | 录制任务ID |
> | eventData.recordState | object | 录制状态信息 |
> | eventData.recordState.startTs | number | 录制开始时间戳（毫秒，2000/2001 事件携带） |
> | eventData.recordState.fileInfo | array | 录制文件信息列表（2001 事件携带） |
> | eventData.recordState.fileInfo[].filePath | string | 文件路径 |
> | eventData.recordState.fileInfo[].fileDuration | number | 文件时长（毫秒） |
> | eventData.recordState.fileInfo[].fileSize | number | 文件大小（字节） |
> | eventData.recordState.fileInfo[].timestamp | number | 文件生成时间戳 |
> | eventData.recordState.fileInfo[].status | number | 文件状态，0 为成功 |
> | eventData.recordState.reason | string | 失败原因（2002 事件携带） |
> | eventData.recordState.code | number | 失败错误码（2002 事件携带） |
>
> **返回参数：**
>
> | 参数名 | 类型 | 说明 |
> | --- | --- | --- |
> | code | number | 状态码，200为成功 |
> | message | string | 返回消息 |
>
> **返回参数示例（成功）：**
>
> ```json
> {
>   "code": 200,
>   "message": "success"
> }
> ```
>
> **返回参数示例（签名验证失败）：**
>
> ```json
> {
>   "code": 403,
>   "message": "签名验证失败"
> }
> ```

#### 3.4.1 查询会议信息
